## Why FastRobot?

`java.awt.Robot` is convenient but slow. Its screen capture is bottlenecked by Java2D (15-30fps max), and input has OS-level throttling.

FastRobot solves this with:
- **GDI BitBlt** for ultra-fast screen capture (60fps+ capable) ‚ÄĒ see [Performance Benchmarks](#performance-benchmarks)
- **DirectInput** for zero-latency keyboard/mouse events ‚ÄĒ see [FastRobot vs java.awt.Robot](#fastrobot-vs-javaawtrobot)
- **Native memory buffers** to avoid GC overhead
- **Hardware acceleration** via [DXGI Desktop Duplication API](https://docs.microsoft.com/en-us/windows/win32/direct3ddxgi/desktop-dup-api)
- **Drop-in API** ‚ÄĒ minimal code changes required

## Quick Start

### Basic Usage

```java
import fastrobot.FastRobot;
import java.awt.Rectangle;
import java.awt.image.BufferedImage;

FastRobot robot = new FastRobot();

// Fast screen capture - 10-17√ó faster than Robot
BufferedImage screen = robot.createScreenCapture(new Rectangle(0, 0, 1920, 1080));

// Instant mouse control - 515√ó faster latency
robot.mouseMove(500, 500);
robot.mousePress(FastRobot.BUTTON1);
robot.mouseRelease(FastRobot.BUTTON1);

// Low-latency keyboard
robot.keyPress(java.awt.event.KeyEvent.VK_A);
robot.keyRelease(java.awt.event.KeyEvent.VK_A);
```

### High-FPS Streaming

```java
// Start 60fps-240fps streaming capture
robot.startScreenStream(0, 0, 1920, 1080);

// Non-blocking frame retrieval
while (true) {
    if (robot.hasNewFrame()) {
        int[] pixels = robot.getNextFrame(); // RGBA pixel array
        // Process frame in real-time
        double fps = robot.getStreamFPS();
        System.out.println("Streaming at " + fps + " FPS");
    }
}

// Stop streaming
robot.stopScreenStream();
```

## Architecture

FastRobot uses a hybrid Java + native architecture:

```
Java API (FastRobot.java)
    ‚Üď JNI
Native Layer (C++/Win32)
    ‚Ēú‚ĒÄ‚ĒÄ DirectInput ‚Üí Mouse/Keyboard (zero latency)
    ‚Ēú‚ĒÄ‚ĒÄ GDI BitBlt ‚Üí Screen capture (fast sync)
    ‚ĒĒ‚ĒÄ‚ĒÄ DXGI Desktop Duplication ‚Üí Streaming (60-240fps async)
    ‚Üď
Windows OS (Hardware)
```

**Key technologies:**
- **JNI** ‚ÄĒ Java-to-native bridge
- **DirectInput** ‚ÄĒ Direct hardware access for input
- **GDI BitBlt** ‚ÄĒ Fast synchronous screen capture
- **DXGI Desktop Duplication API** ‚ÄĒ Hardware-accelerated streaming
- **Native memory buffers** ‚ÄĒ Zero GC pressure

## API Reference

### Mouse Operations
- `mouseMove(int x, int y)` ‚ÄĒ Instant cursor positioning
- `mousePress(int buttons)` ‚ÄĒ Direct hardware button press
- `mouseRelease(int buttons)` ‚ÄĒ Direct hardware button release
- `mouseWheel(int wheelRotation)` ‚ÄĒ Precise wheel control

### Keyboard Operations
- `keyPress(int keycode)` ‚ÄĒ Zero-latency key press
- `keyRelease(int keycode)` ‚ÄĒ Zero-latency key release

### Screen Capture
- `getPixelColor(int x, int y)` ‚ÄĒ Single pixel (100√ó faster)
- `getScreenPixels(int x, int y, int w, int h)` ‚ÄĒ Region capture (int[])
- `createScreenCapture(Rectangle rect)` ‚ÄĒ BufferedImage capture

### v2.0 Streaming (High-FPS)
- `startScreenStream(int x, int y, int w, int h)` ‚ÄĒ Begin 60fps+ capture
- `getNextFrame()` ‚ÄĒ Get next frame (non-blocking, returns int[])
- `getNextFrameScaled(int w, int h)` ‚ÄĒ Get scaled frame (C++ nearest-neighbor)
- `getNextFrameScaledAA(int w, int h, boolean useAA)` ‚ÄĒ Get scaled frame with optional box-filter AA
- `hasNewFrame()` ‚ÄĒ Check if new frame available
- `getStreamFPS()` ‚ÄĒ Get current streaming FPS
- `stopScreenStream()` ‚ÄĒ Stop and cleanup

### Screen Info
- `getScreenWidth()` ‚ÄĒ Primary monitor width
- `getScreenHeight()` ‚ÄĒ Primary monitor height

## Limitations

FastRobot is extremely fast ‚ÄĒ but intentionally focused:

- **Windows-only** ‚ÄĒ requires DirectX 11+ and Windows 10/11
- **Not cross-platform** ‚ÄĒ Linux and macOS are not supported
- **Not suitable for anti-cheat-protected games** ‚ÄĒ screen capture is detectable
- **Requires GPU support** ‚ÄĒ Desktop Duplication API needs compatible graphics
- **Capture performance depends on monitor resolution** ‚ÄĒ 4K capture is slower than 1080p
- **Single monitor only** ‚ÄĒ multi-monitor setups capture primary display only

## FAQ

### Is FastRobot faster than Java Robot?
**Yes.** FastRobot is 10-17√ó faster for screen capture and 515√ó faster for mouse input using native DirectInput and GDI BitBlt.

### Does FastRobot use DirectX?
**Yes.** FastRobot v2.0 uses the **DXGI Desktop Duplication API** for hardware-accelerated screen streaming at 60fps+ (matches your monitor's refresh rate).

### Can I use FastRobot for game bots?
**Yes.** FastRobot is designed for low-latency automation with DirectInput for instant mouse/keyboard control, ideal for gaming bots.

### Is FastRobot cross-platform?
**No.** FastRobot is Windows-only (Windows 10/11) to maximize performance using native Win32 APIs.

### How do I migrate from java.awt.Robot?
Replace `new Robot()` with `new FastRobot()` ‚ÄĒ the API is mostly compatible. See [API Reference](#api-reference) for details.
